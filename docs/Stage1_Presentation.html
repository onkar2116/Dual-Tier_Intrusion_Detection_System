The Problem

Problem Statement
❌ Traditional (Signature-Based) IDS
  • Pattern matching against known attack signatures
  • Cannot detect zero-day / unknown attacks
  • Requires constant manual rule updates
  • Easily bypassed with minor attack variations
IF (SYN_count > 100 AND ACK < 5)
THEN attack = "SYN Flood"
⚠ What if attacker sends 99 SYNs? → MISSED!
⚠ ML-Based IDS — Vulnerable to Adversarial Attacks
  • Learns patterns → detects unknown attacks ✓
  • Achieves ~95% accuracy on clean data ✓
  • Fooled by tiny perturbations ✗
Clean → ML Model → ✓ Detected (95%)
Adversarial → ML Model → ✗ Missed (40%)
🎯 Core Problem

Attackers craft adversarial perturbations — tiny invisible changes to network traffic — that cause ML classifiers to misclassify attacks as normal traffic. Existing IDS systems have no defense against this.

Our Solution — Three-Tier Defense Architecture

Defense-in-Depth
Tier 1
Signature-Based Detection
Fast pattern matching against known attack signatures. Database of DoS, Probe, R2L, U2R, brute-force rules.
100% on Known Attacks < 0.5ms/sample
↓  if no signature match
Tier 2
ML Classification (DNN / CNN / LSTM)
Three deep learning models trained on NSL-KDD to detect unknown & novel attacks across 5 categories.
~95% Clean Accuracy ~12ms/sample
↓  if attack detected
Tier 3
Adversarial Defense
Adversarial training + feature squeezing + ensemble voting to detect and resist adversarial evasion.
88% Robust Accuracy ~35ms/sample
💡 Key Innovation

We combine all three layers into a single pipeline where each tier compensates for the weakness of the previous one. No prior IDS integrates all three strategies together.

What Each Tier Handles
Threat Type T1 T2 T3
Known Attacks✓✓✓
Zero-Day Attacks✗✓✓
Adversarial Evasion✗✗✓

Dataset — NSL-KDD

Primary Dataset

Overview

NSL-KDD is the standard benchmark dataset for IDS research, published by the Canadian Institute for Cybersecurity (UNB). It contains labeled network connection records with 41 features per sample.

125K
Training Samples
41
Raw Features
5
Attack Classes

Attack Categories

Class 0 Normal — Legitimate benign traffic
Class 1 DoS — neptune, smurf, back, teardrop
Class 2 Probe — portsweep, ipsweep, nmap, satan
Class 3 R2L — guess_passwd, ftp_write, imap
Class 4 U2R — buffer_overflow, rootkit, loadmodule

Class Distribution Challenge

Normal — 67,34353.4%
DoS — 45,92736.5%
Probe — 11,6569.3%
R2L — 9950.8%
U2R — 520.04%
⚠ Key Challenge — Severe Imbalance

U2R class has only 52 samples out of 125K. A naive model ignoring U2R gets 99% accuracy — but detects zero user-to-root attacks!

✅ Our Fix

SMOTE oversampling + WGAN-GP synthetic generation equalizes all classes before training, forcing the model to learn all attack types equally.

Adversarial Attacks We Implement

Stage 3 Preview

We implement 4 adversarial attack methods to stress-test the system and validate our Tier 3 defense. Each adds carefully crafted noise to trick the ML classifier.

⚡ FGSM — Fast Gradient Sign Method

Single-step gradient attack. Fast but moderate strength.

x_adv = x + ε · sign(∇x Loss)
1 stepFastε = 0.1
🔁 PGD — Projected Gradient Descent

Iterative FGSM with projection. Strongest white-box attack.

x_adv = Πε(x_adv + α · sign(∇ Loss))
40 stepsStrong
🎯 C&W — Carlini & Wagner L2

Optimization-based. Finds minimum perturbation needed to misclassify.

min ||δ||₂  s.t.  f(x+δ) ≠ true_class
100 stepsStrongest
🌀 DeepFool

Finds the closest decision boundary with minimal perturbation.

50 stepsEfficient
