1. Introduction

1.1 Background

Network Intrusion Detection Systems (IDS) serve as a critical line of defense in modern cybersecurity infrastructure. They monitor network traffic in real time to identify malicious activities such as denial-of-service (DoS) attacks, unauthorized access attempts, and data exfiltration. As cyber threats continue to grow in volume and sophistication, the need for intelligent, adaptive detection systems has never been greater.

Traditional IDS solutions rely on signature-based detection — matching incoming traffic against a database of known attack patterns. While effective against previously documented threats, these systems are fundamentally unable to detect novel, unknown, or zero-day attacks. This critical limitation has driven the adoption of machine learning (ML) techniques, which can learn complex patterns from labeled network traffic data and generalize to unseen attack variants.

However, recent research in adversarial machine learning has revealed a significant vulnerability: ML models can be fooled by adversarial examples — carefully crafted inputs with imperceptible perturbations that cause the classifier to produce incorrect predictions. In the context of IDS, an attacker could add tiny modifications to malicious network traffic that cause the ML model to classify it as benign, effectively evading detection entirely.

1.2 Problem Statement

Core Problem

Existing intrusion detection systems face a fundamental security gap: signature-based systems cannot detect unknown attacks, while ML-based systems are vulnerable to adversarial evasion attacks. There is no widely adopted IDS architecture that effectively addresses both limitations simultaneously.

Specifically, the problems are:

  1. Signature-Based IDS Limitations: Cannot detect zero-day attacks, require constant manual updates, and are easily bypassed with minor attack variations.
  2. ML-Based IDS Vulnerability: Deep learning classifiers achieve high accuracy (~95%) on clean data but accuracy drops to as low as 40% under adversarial perturbation attacks such as FGSM and PGD.
  3. Class Imbalance: Real-world network traffic datasets are heavily imbalanced. In the NSL-KDD dataset, the U2R (User-to-Root) class contains only 52 samples out of 125,973, causing models to ignore rare but critical attack types.

1.3 Motivation

The increasing deployment of ML models in security-critical applications demands that these systems be robust against adversarial manipulation. A successful adversarial attack against an IDS could have catastrophic consequences — allowing attackers to penetrate enterprise networks, healthcare systems, financial infrastructure, and critical national infrastructure undetected.

Key motivating factors include:

  • 43% increase in network intrusion attempts reported globally in recent years
  • Average cost of a data breach: $4.45 million (IBM Security Report, 2023)
  • Growing sophistication of Advanced Persistent Threats (APTs) using AI-powered evasion
  • NIST and IEEE guidelines emphasizing the need for adversarial robustness in ML systems

1.4 Objectives

The primary objectives of this project are:

  1. Design and implement a three-tier IDS architecture that combines signature-based detection, deep learning classification, and adversarial defense mechanisms.
  2. Train and evaluate three deep learning architectures — DNN, CNN, and LSTM — for multi-class network traffic classification.
  3. Implement four adversarial attack methods — FGSM, PGD, C&W, and DeepFool — to evaluate model vulnerability.
  4. Develop three defense strategies — adversarial training, input transformation (feature squeezing), and ensemble voting — to improve robust accuracy.
  5. Build a WGAN-GP module for synthetic attack traffic generation and data augmentation.
  6. Create an interactive Streamlit dashboard for real-time monitoring, batch analysis, and attack simulation.
  7. Achieve >95% clean accuracy and >85% robust accuracy against adversarial attacks.

2. Literature Review

2.1 Traditional IDS Approaches

Intrusion Detection Systems are broadly categorized into signature-based (misuse detection) and anomaly-based systems. Signature-based IDS, such as Snort and Suricata, match network packets against a database of known attack patterns. These systems provide high precision for known threats with virtually zero false positives but cannot identify novel attacks not present in the signature database.

Anomaly-based systems establish a baseline of "normal" behavior and flag deviations. While capable of detecting unknown attacks, they suffer from high false positive rates, making them impractical for production deployment without significant tuning.

2.2 ML-Based Intrusion Detection

Machine learning approaches have emerged as a powerful alternative for IDS, leveraging supervised classification to categorize network traffic. Notable works include:

  • Buczak & Guven (2016) — Comprehensive survey of data mining and ML methods for cybersecurity IDS, establishing benchmarks on the NSL-KDD dataset.
  • Ahmad et al. (2021) — Systematic study demonstrating that deep neural networks outperform traditional ML algorithms (Random Forest, SVM) for multi-class attack classification.
  • 1D-CNN approaches — Treating network features as sequential data, achieving superior pattern recognition for DoS and Probe attacks.
  • LSTM-based approaches — Capturing temporal dependencies in network traffic flows, particularly effective for slow-rate attacks.

2.3 Adversarial Machine Learning

The vulnerability of deep neural networks to adversarial examples was first demonstrated by Goodfellow et al. (2015) with the Fast Gradient Sign Method (FGSM). Subsequent work has produced increasingly sophisticated attacks:

Attack Method Authors Year Key Contribution
FGSM Goodfellow et al. 2015 Single-step gradient-based attack; fast but moderate strength
PGD Madry et al. 2018 Iterative FGSM with projection; strongest first-order attack
C&W Carlini & Wagner 2017 Optimization-based; finds minimum perturbation for misclassification
DeepFool Moosavi-Dezfooli et al. 2016 Computes minimal perturbation to cross decision boundary

Defense mechanisms include adversarial training (training on adversarial examples), input transformation (preprocessing inputs to remove perturbations), and ensemble methods (aggregating predictions from multiple models). Madry et al. (2018) demonstrated that PGD-based adversarial training provides empirically robust models, while Xu et al. (2018) introduced feature squeezing as a lightweight detection mechanism.

2.4 Generative Adversarial Networks for IDS

GANs, introduced by Goodfellow et al. (2014), learn to generate realistic synthetic data. In the IDS context, GANs address the critical class imbalance problem by generating synthetic samples of rare attack types. The Wasserstein GAN with Gradient Penalty (WGAN-GP), proposed by Gulrajani et al. (2017), provides stable training without mode collapse, making it suitable for generating network traffic data.

3. Proposed System Architecture

3.1 Three-Tier Defense Overview

Our system follows a defense-in-depth principle, where each tier addresses a specific class of threats that the previous tier cannot handle. This layered approach ensures comprehensive coverage against known attacks, unknown attacks, and adversarial evasion attempts.

Tier Component Threat Coverage Speed Key Technology
Tier 1 Signature Detection Known attacks (100%) < 0.5ms JSON rule database, pattern matching
Tier 2 ML Classification Unknown/novel attacks ~12ms TensorFlow DNN/CNN/LSTM
Tier 3 Adversarial Defense Adversarial evasion ~35ms PyTorch adversarial training

3.2 Tier 1: Signature-Based Detection

Tier 1 provides fast, deterministic detection of known attack patterns using a JSON-based signature database. Each signature defines a set of conditions (feature thresholds) that characterize a specific attack type. The matching algorithm computes a match ratio and triggers detection when the ratio exceeds a configurable threshold (default: 0.8).

Signature Database Categories:

  • DoS Attacks: SYN Flood, UDP Flood, HTTP Flood (Slowloris)
  • Port Scanning: TCP Port Scan
  • Brute Force: SSH Brute Force, FTP Brute Force
  • Web Attacks: SQL Injection, Cross-Site Scripting (XSS)
  • Botnet: C2 Communication patterns
  • NSL-KDD Specific: Neptune, Smurf, Port Sweep, Satan, Buffer Overflow

Matching Algorithm:

// Pseudocode for signature matching
for each signature in database:
    match_count = 0
    total_conditions = len(signature.conditions)

    for each (feature, rule) in conditions:
        if rule is range:    check min/max bounds
        if rule is list:     check value membership
        if rule is scalar:   check exact equality
        if matched: match_count += 1

    match_ratio = match_count / total_conditions
    if match_ratio >= 0.8: return ATTACK DETECTED

3.3 Tier 2: ML Classification

Tier 2 employs three deep learning architectures trained to classify network traffic into five categories: Normal, DoS, Probe, R2L, and U2R. All three models are trained and the best-performing model (by validation accuracy) is selected for deployment.

3.3.1 Deep Neural Network (DNN)

Four-layer fully connected network optimized for tabular network traffic data:

Input(n_features) → Dense(256, ReLU) → BatchNorm → Dropout(0.3)
                  → Dense(128, ReLU) → BatchNorm → Dropout(0.3)
                  → Dense(64,  ReLU) → BatchNorm → Dropout(0.3)
                  → Dense(32,  ReLU) → Dense(5, Softmax) → Output

3.3.2 1D Convolutional Neural Network (CNN)

Treats features as a 1D sequence, using convolutional layers to detect local patterns:

Input(n_features, 1) → Conv1D(64, k=3) → BN → MaxPool
                     → Conv1D(128, k=3) → BN → MaxPool
                     → Conv1D(256, k=3) → BN → Flatten
                     → Dense(128) → Dropout → Dense(64) → Softmax

3.3.3 LSTM Network

Captures sequential dependencies in traffic features:

Input(1, n_features) → LSTM(128, return_seq) → Dropout(0.3)
                     → LSTM(64) → Dropout(0.3)
                     → Dense(32, ReLU) → Dense(5, Softmax)

3.4 Tier 3: Adversarial Defense

Tier 3 implements three complementary defense strategies to detect and resist adversarial evasion attacks:

Strategy 1: Adversarial Training

During training, each batch is composed of 40% clean samples, 30% FGSM-perturbed samples, and 30% PGD-perturbed samples. The model learns to classify correctly regardless of whether the input has been adversarially perturbed.

Strategy 2: Input Transformation (Feature Squeezing)

At inference time, the input is passed through a bit-depth reduction filter (16 quantization levels). Predictions on the original and squeezed inputs are compared. If the L1 distance between predictions exceeds a threshold (0.1), the input is flagged as adversarial.

Strategy 3: Ensemble Voting

Multiple models (DNN, CNN, LSTM) independently predict probability distributions. Predictions are averaged via soft voting. Low agreement among models indicates a potential adversarial input, as adversarial perturbations often do not transfer equally across architectures.

3.5 GAN Module (WGAN-GP)

A Wasserstein GAN with Gradient Penalty generates realistic synthetic attack traffic for data augmentation and stress testing. The generator transforms 100-dimensional noise vectors into feature vectors matching the dimensionality of preprocessed network traffic. The discriminator uses a 5:1 training ratio (5 discriminator updates per generator update) with gradient penalty (weight = 10) for training stability.

The NSL-KDD dataset is a refined version of the original KDD Cup 1999 dataset, published by the Canadian Institute for Cybersecurity at the University of New Brunswick. It addresses key issues in the original dataset such as redundant records and class overlap, making it the standard benchmark for IDS research.

Critical Rule: No Data Leakage

The scaler is fitted exclusively on training data and then applied (transformed) to validation and test sets. SMOTE is applied only to training data. This prevents information from the test set leaking into the training process and ensures evaluation metrics reflect true generalization performance.

8. Adversarial Attack Methods

We implement four adversarial attack methods to comprehensively evaluate the robustness of our ML models. These attacks generate adversarial examples by adding carefully computed perturbations to the input, causing the classifier to mispredict.

8.1 FGSM — Fast Gradient Sign Method

Proposed by Goodfellow et al. (2015), FGSM computes a single-step perturbation in the direction of the loss gradient:

# Mathematical formulation
x_adv = x + ε · sign(∇x L(f(x), y))

# Implementation (src/adversarial_attacks/fgsm.py)
def fgsm_attack(model, x, y, epsilon=0.1):
    x_adv = x.clone().detach().requires_grad_(True)
    output = model(x_adv)
    loss = F.cross_entropy(output, y)
    loss.backward()
    perturbation = epsilon * x_adv.grad.sign()
    x_adv = torch.clamp(x + perturbation, 0, 1)
    return x_adv.detach()

8.2 PGD — Projected Gradient Descent

Proposed by Madry et al. (2018), PGD is an iterative version of FGSM that applies multiple smaller steps and projects back into the epsilon-ball after each step. It is considered the strongest first-order attack:

# Iterative attack: 40 steps, step size α = 0.01, ε = 0.1
for i in range(num_iterations):
    x_adv.requires_grad_(True)
    loss = F.cross_entropy(model(x_adv), y)
    loss.backward()
    x_adv = x_adv + α * x_adv.grad.sign()       # Take step
    perturbation = clamp(x_adv - x, -ε, ε)      # Project
    x_adv = clamp(x + perturbation, 0, 1)         # Valid range

8.3 C&W — Carlini & Wagner L2 Attack

An optimization-based attack that finds the minimum perturbation required for misclassification. It solves: minimize ||δ||2 subject to f(x + δ) ≠ true_class. Implemented via IBM's Adversarial Robustness Toolbox (ART).

8.4 DeepFool

Iteratively computes the minimal perturbation needed to cross the nearest decision boundary. Produces smaller, more efficient perturbations than FGSM or PGD.
